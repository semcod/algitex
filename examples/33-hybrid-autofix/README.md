# Example 33: Hybrid AutoFix

Fast parallel mechanical fixes + rate-limited LLM fixes in one workflow.

## What is Hybrid AutoFix?

Combines two execution strategies:

| Phase | Type | Speed | LLM? | Use Case |
|-------|------|-------|------|----------|
| Phase 1 | Mechanical | ~1500 tps | ❌ No | Unused imports, return types |
| Phase 2 | LLM | ~10-50 tps | ✅ Yes | Complex fixes requiring reasoning |

**Key Features:**
- Rate limiting (token bucket algorithm)
- Retry logic with exponential backoff
- Cost tracking per LLM call
- Parallel execution with conflict-free per-file isolation

## Usage

```bash
# Preview (dry run)
python main.py --demo dry-run

# Verify tasks first
python main.py --demo verify

# Benchmark performance
python main.py --demo benchmark

# Fix only mechanical issues (fast)
python main.py --demo mechanical --execute

# Full hybrid with LiteLLM proxy
python main.py --demo hybrid --execute --workers 4 --rate-limit 10

# 100% offline with Ollama
python main.py --demo ollama --execute --workers 2 --rate-limit 5
```

## API Example

```python
from algitex.todo import HybridAutofix

fixer = HybridAutofix(
    backend="litellm-proxy",
    proxy_url="http://localhost:4000",
    workers=4,
    rate_limit=10,      # 10 LLM calls per second
    retry_attempts=3,
    dry_run=False       # Actually execute
)

# Phase 1: Parallel mechanical fixes
mechanical_result = fixer.fix_mechanical("TODO.md")

# Phase 2: Rate-limited LLM fixes
llm_result = fixer.fix_complex("TODO.md")

# Or both at once
result = fixer.fix_all("TODO.md")
fixer.print_summary(result)
```

## Backend Options

| Backend | Description | Best For |
|---------|-------------|----------|
| `litellm-proxy` | Proxy to multiple providers | Production, cost tracking |
| `ollama` | Local LLM (100% offline) | Privacy, zero API costs |
| `aider` | Aider AI code editor | Complex refactoring |

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│              HybridAutofix                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Phase 1: Mechanical                                    │
│  ├── Parse TODO.md                                      │
│  ├── Filter: unused_import, return_type                 │
│  ├── ThreadPoolExecutor(8 workers)                     │
│  └── Apply regex-based fixes                           │
│                                                         │
│  Phase 2: LLM                                           │
│  ├── Filter: fstring, magic, docstring, exec_block     │
│  ├── RateLimiter(rate=10/sec)                          │
│  ├── ThreadPoolExecutor(workers)                       │
│  └── Call LLM backend with retry logic                 │
│                                                         │
│  Summary: fixed, skipped, errors, cost, time           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## Requirements

- LiteLLM proxy running (for litellm-proxy backend): `make proxy`
- Or Ollama running (for ollama backend): `ollama serve`
- TODO.md file generated by prefact
